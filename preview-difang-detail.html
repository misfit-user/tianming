界 面 设 计 说 明

〔 设 计 要 点 〕

① 立绘区 130×170 — 默认空状态：渐变 + 卷宗式角标 + 大写首字 + 楷体提示「点击导入立绘」；玩家点击触发 file-input，导入 png/jpg/webp/gif，base64 存入 localStorage（key 按 div.id），刷新立即生效；hover 时浮出 [更换] [移除] 按钮。

② Hero 角部加四个铜框角标（::before/::after）— 远观如卷轴边框；右侧名号识别条按层级排布：大字标题 → meta pills 行（级别/regionType/地形/税额/官职/主官） → 折叠描述 → 特产 → 6 个 tag 徽章。

③ 6 格快览（户口/耕地/商业/实征/民心/腐败）— 一眼可知此区轻重缓急；hover 时上浮 1px。

④ 各 section 标题统一升级：左侧图标 + 楷体粗字 + 底部双线（rgba 横线 + 36px 渐变金线下划），扫读节奏明显。

⑤ 经济基础卡片仅按 tag 显示对应字段（北直隶有海贸/盐/矿；浙江仅海贸/渔；福建仅海贸/渔），避免冗余 0 占位。

⑥ 朱印色（vermillion）专留给皇室直辖资产 — 视觉上一眼区分「朝廷直管」与「布政司征税」。

⑦ 整体宽度 780px（覆盖原 600px 默认）— `:has()` 选择器仅作用于此 modal，其他通用 modal 不受影响。

⑧ 应输上解·赋税核算 — 按 division.economyBase × policies 实时算 8-9 项税收（田赋·正/附加 / 丁银 / 漕粮折银 / 盐课 / 矿税 / 关津税 / 城商税 / 市舶 / 渔课 / 马征折银），每行展示 name | formula | amount，公式透明可审。底部合计 + 与 div.fiscal.claimedRevenue 实绩比对（颜色按比例：>90%绿 / 60-90%金 / <60%朱）。

⑨ 应负岁出·经费核算 — 仅展示「可归属本区」的支出：驿递站银（postRelays × 200）/ 教育常费（kejuQuota × 50）/ 在地俸禄（GM.chars filter location 之 salary 之和）/ 在地军饷（GM.armies filter station × payPerSoldier × 12）/ 本回合赈灾水利（div.fiscal.expenditures）。每行同样给出公式。

⑩ 在灾实录 — 读 economyBase.disasterRecord[]·6 类灾种(旱/水/瘟/蝗/震/寒) × 3 档烈度 × 起始回合 × 备注；无在灾时若有 ecoScars 显示「生态痤疮」提示。

⑪ 公库三账·细目 — 三账横排卡片(银/粮/布)·每张含 stock + quota 进度条 + 占比 + 入/出流量 + 亏空徽标 + 现掌库/前任/交接案卷数。

⑫ 词条改名：户口 → 在编户口·耕地 → 在编田亩（强调 fugitives/hidden 之外的可征收基础）。

⑬ 田亩 default 改地形系数：原 户 × 30 一刀切 → 新 户 × 系数(terrain)。系数表（参《万历会计录》各省实田 ÷ 黄册户）：平原 32 / 盆地 24 / 丘陵 22 / 沿海 16 / 高原 14 / 山地 9 / 草原 4 / 荒漠 2。北直隶 4730 → 5046 (历史 4910-6000) ✓·浙江 5307 → 3892 (历史 4633) ✓·福建 2769 → 1477 (历史 1369) ✓·四川 4327 → 由盆地 24 系数算 1442×24=3461 (历史 1342·黄册严重低报)。皇庄 default 由 farmland × 10% → 5%·更保守。

⑭ 实绩比对·双向警示：原 ratio >90% 单方向判健康 → 新区间 85-120%绿(健康) / 60-160%金(偏离) / 其他朱(异常)·朱区附「漏征严重 / 加派重压」标签。浙江 360% 即典型「江南重赋 + 三饷加派」实历。
