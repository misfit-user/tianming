# 常朝 v3 · console 测试操作手册

**目标**：在浏览器控制台启用 v3·跑通完整朝议流程·把发现的 bug 反馈给开发者迭代。

**前置条件**：v3 已加载（看 console·应有 `_cc3_open` 全局函数）·主项目已配 API（设置面板）·游戏已开新档进入回合。

---

## §1 启用 v3

```js
window.USE_CC3 = true
```

预期：无报错·console 不刷红字。

随时回退到 v2：
```js
window.USE_CC3 = false
```

---

## §2 打开早朝

### 方式 A · 走主游戏入口（最接近真实）
1. 顶栏点击 **🏛 朝议** 按钮（FAB 或 鸿雁面板）
2. 弹三模式选卡·点 **📜 常朝**
3. 应进入 v3·看到 v3 独立 modal（不是 v2 的 chaoyi-modal）

### 方式 B · 直接调函数（绕过菜单）
```js
_cc3_open()
```

**预期效果**：
- 屏幕中央弹出 v3 modal（深色宣纸纹理·标题"〔早朝〕"）
- 班次区显示真实 GM.chars 中的角色（不再是 mock 韩爌等）
- 标题栏右侧 🕒 时辰指示
- 主舞台开始播鸣鞭三响 → 山呼万岁 → 缺朝名册 → 御殿
- 决断区显示"奏来 / 此事免议 / 改日再奏"三按钮

## §3 完整流程测试矩阵

### T1 · 基础流程（10 分钟）
1. 走完一条议程的"启奏 → 详述 → 决断"
2. 点准奏·看主奏者+派系反应
3. 退朝·看朝会总结面板的圆胶囊 tally

**通过标准**：每个阶段都有 NPC 发言·没有"undefined"·没有空白气泡

### T2 · 玩家自由话语（5 分钟）
1. 详述阶段·输入框打 "陕西饥情甚重 朕忧之"
2. 看主奏者+1 NPC 立场化回应（带派系倾向）
3. 输入"温体仁可斩否？"·应触发温体仁抢答（如该角色在场）

**通过标准**：NPC 回应紧扣玩家话内容·不是模板

### T3 · 关键词自动触发（2 分钟）
- 详述阶段输 "准" → 自动准奏
- 详述阶段输 "驳" → 自动驳奏
- 详述阶段输 "留中" → 自动留中
- 详述阶段输 "下廷议" → 自动转廷议

**通过标准**：关键词识别准·不需要点按钮

### T4 · 朝威分流（5 分钟）
1. 浏览器 console 临时改皇威皇权（如 console 跑 `GM.vars['皇威'].value=80; GM.vars['皇权'].value=80`）
2. 跑 `_cc3_close()` 然后 `_cc3_open()` 重开
3. 详述阶段后·rank ≥3 NPC 应入"请奏队列"·决断区出现 `📋 请奏 N 人 ▼` 按钮
4. 拉低皇威皇权（< 75）·重开·自发表态恢复

**通过标准**：高朝威 = 肃朝（请奏队列）·低朝威 = 众言（自发表态）

### T5 · 抗辩面板（条件触发·有运气）
1. 找 controversial ≥ 7 的议程（弹劾类大概率）
2. 准奏后 45% 概率某反方 NPC 出列严辞抗辩
3. 试三选：[🎤 听其抗辩] / [🛡️ 朕意已决] / [⚡ 严斥]
4. 听其抗辩 → [📝 从其议] 看决议反转

**通过标准**：抗辩流程能跑通·不卡死

### T6 · 严斥 5 outcomes（5 分钟）
1. 决断区点 ⋯ 更多 → 训诫 → 选某 NPC
2. 看 5 种结局之一（叩首/面服心怨/伏阙请辞/暗结同党/当庭抗辩）

**通过标准**：5 种结局每种都有专色徽标显示·loyalty/数值变化反馈

### T7 · 时辰流动（边走边看）
- 第 1 议程：标题栏 🕒 五更三点
- 第 4 议程：寅时三刻
- 第 7 议程：卯时四刻

**通过标准**：议程推进·时辰跟着推

### T8 · 喧哗（条件触发·罕）
1. 高 controversial 议程进议论·点续议一轮
2. 议论第 2 轮 + controversial ≥ 8 → 整个 modal 朱红光晕脉冲
3. 决断区出现 `🔔 鸣磬肃静` 红色按钮·点击 → 红雾消失

**通过标准**：喧哗触发条件准·肃静按钮可清

### T9 · 退朝总结（最后一步）
- 走完所有议程或点退朝
- 弹"〔朝会已散〕"面板·tally 圆胶囊（准N/驳N/留N）+ 议程明细分色
- 勾"下次不再弹"·下次开早朝不弹此面板

**通过标准**：tally 数字与你的实际操作匹配

### T10 · 切回 v2 验证（最后兜底）
```js
window.USE_CC3 = false
```
重开早朝·应回到 v2 旧 UI。

**通过标准**：切回干净·v2 不破坏

## §4 常见问题排查

| 现象 | 原因猜测 | 临时处置 |
|---|---|---|
| `_cc3_open` 报 undefined | v3 没加载 | F5 刷新 + 检查 console 有无加载错误 |
| 班次区是空 | GM.chars 是空或都不在京 | console 跑 `GM.chars.length` / `GM.chars.filter(c => _isAtCapital(c)).length` 看数据 |
| 议程是"日常无事·内侍" | AI 议程生成失败·走 fallback | 看 console 有无 [tm-chaoyi-v3:agenda] error · 通常是 API 配置问题 |
| 玩家说话 NPC 不回应 | aiEnabled() 返回 false | console 跑 `_cc3_aiEnabled()` 应返回 true·若 false 检查 `P.ai.key` `P.ai.url` |
| modal 错位/样式乱 | tm-chaoyi-v3.css 没加载 | console 跑 `document.getElementById('cc3-css')` 检查·或 F12 网络面板看 css 请求 |
| modal 关不掉 | _cc3_close 逻辑漏 DOM | console 跑 `document.querySelectorAll('.cy-stage').forEach(e=>e.remove())` 强清 |
| 卡在"入殿仪礼中" | runOpening 里的 timing bug | F12 console 看 throw·复制错误信息 |

## §6 已知风险点（不算 bug）

1. **CHARS rank/class 推断粗糙**：preview 写死的 1-9 品 / east/wu/kdao 三类·v3 用关键词推（"将军/总兵→wu" / "御史/给事中→kdao"）·可能误归类。校准在波 2。
2. **AGENDA 字段不全**：preview 期望 selfReact / debate2 / target 等·真 AI 议程生成可能没吐这些·所以朝议会"卡"在某些阶段。波 2 改 prompt schema。
3. **modal 共用 DOM id**：v3 用 `cy-stage` 等 id·与 v2 chaoyi-modal 内部 id 部分同名。理论上 v2/v3 modal 不同时存在·但快速切换可能残留。`_cc3_close` 已加防护。
4. **召唤/角色数据未接**：`_cc3_callAI` 的 prompt 还没融合 v2 character/personality/loyalty 上下文·NPC 回应较扁平。波 2 修。
